# PRD: Garden Overview App

## 1. Purpose
The application is intended to provide the user with a simple and visual way to plan, organize and follow up on their own garden throughout the season. It combines maps (garden map), planning (what and where), and follow-up (when things are actually done).

---

## 2. Target group
- Hobby gardeners
- People with kitchen gardens, pallet frames or smaller gardens
- Users who want structure without using complicated systems

---

## 3. Problem
Users lack:
- Overview of what is planted where
- Structure of sowing, planting and harvesting
- History from previous seasons
- An easy way to combine visual plan and calendar

---

## 4. Solution description
An application that combines:
- A visual garden map with layers
- Planning tool for planting and sowing
- Registration of activities
- Calendar and tasks
- History per season

---

## 5. Core functions

### 5.1 Garden map (Visual overview)
- Drawing the garden in top-down view
- Adding areas:
- Pallet frames
- Beds
- Trees and shrubs
- Clickable areas for detail view

### 5.2 Layers
The user can turn on/off:
- Kitchen garden
- Flowers
- Shrubs and trees
- Planned planting
- Actual planting
- Historical data

---

### 5.3 Planning
For each plant:
- Name
- Type (vegetable, flower, berry)
- Location in the garden
- Sowing method:
- Pre-cultivation inside
- Direct sowing outside

Time data:
- Sowing date
- Planting date
- Direct sowing date
- Expected harvest

---

### 5.4 Follow-up / Logging
The user can register:
- Sown inside
- Direct sowing
- Planted outside
- Watered
- Fertilized
- Harvested

Registration can be done via:
- Map
- List view

---

### 5.5 Calendar and tasks
- Calendar (week/month)
- List of upcoming tasks
- Events based on planned dates

Examples:
- “Plant tomatoes”
- “Fertilize pallet frame 2”
- “Harvest lettuce”

---

### 5.6 Plant profiles
Minimum:
- Name
- Type
- Notes

Extensions:
- Sowing time
- Sun conditions
- Water requirements
- Plant spacing

---

### 5.7 Notes
- Per plant
- Per area
- General season notes

---

### 5.8 History
- See previous seasons
- What was planted where
- Compare years
- Learning over time

---

## 6. User stories

- As a user, I want to draw my garden so that I have an overview
- As a user, I want to enter what to plant where
- As a user, I want to know when to sow and plant
- As a user will I record what I have actually done
- As a user, I will get an overview of what I should do now
- As a user, I will be able to see what I did in previous years

---

## 7. MVP (Minimum Viable Product)

First version should include:
1. Garden map with simple areas
2. Enter what is planted where
3. Distinguish between kitchen garden and flowers (layers)
4. Record:
- Sowed
- Planted out
- Harvested
5. Simple calendar with tasks

---

## 8. Future features
- Reminders (push)
- Pictures per plant/area
- Statistics (yield, success rate)
- Automatic suggestions (based on plant)
- Sharing of garden plan

---

## 9. Success criteria
- The user records data regularly
- The user gets a better overview of the garden
- The user uses the app over several seasons
- Reduced need for manual notes

---

## 10. Open questions
- How detailed should plant profiles be?
- How advanced should the map be?
- Should the app support multiple gardens?
- How much automation is desired?
